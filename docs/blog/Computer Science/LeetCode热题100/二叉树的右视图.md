## 解题思路

这道题是要我们从右往左看二叉树，优先右侧，右侧的节点会遮挡左侧的节点，所以，当我们第一次达到树的某一个新的深度的时候，我们访问的节点必须是这一层最右侧的节点，也就是在右侧能看到的节点。所以，我们可以：

- 总是按照 **根 -> 右 -> 左** 的顺序遍历整棵树；
- 每次往下走一层，深度就加1；
- 比较当前的深度和我们已经找到的右视图节点的数量，如果它们相等，说明我们是第一次来到这个深度，而由于我们是严格按照根->右->左的顺序来遍历的，所以如果我们第一次来到某个深度，遇到的节点一定是这一层最右边的节点，如果`当前深度 == 结果列表的长度` 这个条件，如果为true，说明我们是第一次来到这个深度，这个节点是右视图能看到的节点，将节点值加入到res中，如果为false，说明不是第一次来到这个深度，说明右侧已经都遍历过了，回退到了左侧了，这个时候当前深度也肯定不等于res列表的长度了，就不需要添加节点值到res中，直到遍历结束。

```cpp
/**
 * 作用：获取二叉树的右视图
 * 功能：给定二叉树的根节点，返回从右侧所能看到的节点值集合。
 * * 怎么实现的 (核心逻辑)：
 * 1. 采用深度优先搜索 (DFS)，并严格按照 "根 -> 右 -> 左" 的顺序遍历树。
 * 2. 遍历时携带当前节点的深度 (depth)，初始根节点深度为 0。
 * 3. 维护一个结果数组 res。由于我们优先访问右侧，所以当程序第一次到达某个特定的深度时，
 * 遇到的节点必然是该层最右侧的节点（即没被遮挡的节点）。
 * 4. 判定条件：当 `当前深度 depth == 结果数组的大小 res.size()` 时，说明来到了一个
 * 全新的深度，将当前节点值放入 res。
 * * 优化点：将 res 作为局部变量并按引用传递，避免了类成员变量可能导致的状态污染问题。
 */
class Solution {
public:
    // 递归辅助函数，res 改为引用传递 vector<int>&
    void dfs(TreeNode* root, int depth, vector<int>& res){
        if(root == nullptr){
            return;
        }
        
        // 核心判断：当前深度与收集到的右视图节点数量相等时，说明是新的一层的最右节点
        if(depth == res.size()){
            res.push_back(root->val);
        }
        
        // 优先向右侧深入
        dfs(root->right, depth + 1, res);
        // 右侧到底或遇空后，再向左侧探索
        dfs(root->left, depth + 1, res);
    }

    vector<int> rightSideView(TreeNode* root) {
        vector<int> res; // 将 res 声明为局部变量
        dfs(root, 0, res); // 传入 res 的引用
        return res;
    }
};
```
