# 博客重构记录

## 1. 目标说明

本次重构遵循两个原则：

1. 优先保证当前博客已有功能、页面入口、模板渲染和附件下载不被破坏。
2. 在不大面积推翻现有结构的前提下，先完成一轮低风险、可回溯、可持续的结构收敛。

本次改动重点不是“彻底重写项目”，而是先修正已经确认的问题，包括：

- 缺失资源引用

- hooks 与模板目录职责混杂
- about 相关附件路径分散

## 4. 为什么这样改

这轮改动不是为了“看起来更专业”，而是为了先解决 4 个最现实的问题：

1. 有些页面或配置已经在引用不存在的资源。
2. 同类资源散落在多个位置，后续你自己也难以追踪。
3. 模板、脚本、附件、内容之间缺少边界，项目很容易继续失控。
4. 一些明显的缓存和本地产物没有被隔离，仓库噪声太大。

所以本次重构采用的是“保守收敛法”：

- 先让路径正确
- 再让结构更清楚
- 最后才考虑更大规模的目录重命名和内容分层

### 12.5 为什么这轮不删除源文件

虽然这些 `.tex/.typ/.ai/.fla/.py` 文件并不直接参与页面渲染，但它们依然有保留价值：

- 你后续可能还会继续改图

所以本轮采用的是：

1. 继续把首页从页面内联样式中解耦
2. 继续增强作者页的内容质量与作品集表达能力

### 18.1 将首页样式抽离到独立 CSS

在前一轮首页重构后，`docs/index.md` 的内容结构已经比较清楚，但页面中仍保留了一整段只服务于首页的内联样式。

因此本轮新增：

- `docs/stylesheets/home-page.css`

该文件现在统一承载首页以下模块的样式：

- Hero 区
- 精选内容区
- 代表专题区
- 快速入口区

并在 `docs/index.md` 中接入：

- `stylesheets/home-page.css`

同时移除了原本写在 `index.md` 内部的大段样式块。

这样做的目的：

- 让首页与 `geren.md` 一样，逐步转向“内容文件 + 页面样式文件”分离的方式
- 降低后续继续维护首页时在 Markdown 文件中来回翻找样式的成本
- 让站点页面级样式管理方式更统一

### 18.2 作者页继续增强“研究兴趣与近期目标”

前一轮虽然已经把 `geren.md` 重构成完整作者主页，但内容上仍然更偏“基础版本”，还可以继续更具体。

因此本轮在 `docs/about/geren.md` 中继续补充了：

- `研究兴趣与近期目标`

这一部分不再泛泛写“我关注技术”，而是更明确说明：

- 当前真正持续投入的问题域
- 为什么要把博客做成可复用知识资产
- 近期希望继续补齐哪些能力或表达形态

这样做的目的：

- 让访问者更容易理解你当前真正想做的方向
- 让作者页从“介绍我是谁”进一步升级为“说明我在往哪里走”

### 18.3 作者页新增“持续建设中的内容资产”说明

本轮还补充了：

这一部分重点不是再列技能，而是把你正在长期建设的几类内容资产说明清楚，包括：

- 算法与题解专题
- 科研问题与实验型笔记

这样做的目的：

- 让访问者知道这个博客不是随手更新的内容集合
- 更直观地体现你正在长期建设的主线内容

### 18.4 作者页新增“专题导览”

为了进一步增强作品集感，本轮在 `geren.md` 中新增：

## 19. 第十二轮重构记录

第十二轮重构继续沿着“方向 2 + 方向 3”的思路推进，但这次不再只盯首页与作者页，而是进一步压缩：

- 全站每一页都会承担的固定外部请求
- 无公式页面也会统一下载的大体积脚本
- 全站公共脚本中的前置报错风险

这一轮更偏向“基础加载链路治理”，核心目标是让普通文章页、目录页和栏目页也一起受益，而不是只有首页首屏更快。

### 19.1 关闭 Material 默认远程字体

本次将 `mkdocs.yml` 中的：

- `theme.font.text: Roboto`
- `theme.font.code: Roboto Mono`

调整为：

- `theme.font: false`

这样做的目的：

- 关闭 Material 默认的远程字体加载行为
- 避免所有页面都为 Roboto / Roboto Mono 额外发起网络请求
- 把字体控制权统一收回到本地样式文件

### 19.2 站点图标改为本地资源

本次将 `mkdocs.yml` 中的站点图标从外部 jsDelivr 图片改为：

- `docs/assets/profile/avatar-front.jpg`

这样做的目的：

- 减少页面级公共外链
- 降低 favicon 对外部图床的可用性依赖
- 让首页与作者页已经本地化的视觉资源继续复用

### 19.3 清理 `mkdocs.yml` 中的远程样式依赖

本次移除了当前会在所有页面统一加载的几条远程样式资源，包括：

- Google Fonts
- `lxgw-wenkai-webfont`
- 远程 `link.css`
- 远程 `ziti.css`

这样做的目的：

- 直接减少每页固定 CSS 请求数
- 避免因为第三方字体或样式表波动导致首屏阻塞
- 让全站样式更多依赖本地 CSS 与系统字体栈

### 19.4 将全站正文改为系统字体栈

为了配合前面的远程字体收口，本次继续调整了：

- `docs/stylesheets/extra.css`

新增并统一使用：

- `--site-text-font-stack`
- `--site-code-font-stack`

并将全站正文与代码字体变量改为系统字体栈，保留：

- `PingFangYingFeng`

继续只服务于诗词等需要特殊视觉气质的模块。

这样做的目的：

- 保留特殊模块的设计感
- 让普通正文排版不再依赖外部字体下载
- 让站点在弱网环境下更快进入可阅读状态

### 19.5 将 MathJax 改为按需加载

此前 `mkdocs.yml` 中会在所有页面统一加载：

- `polyfill.io`
- `mathjax@3`

这意味着即使一篇文章完全没有公式，也会承担额外脚本下载与解析成本。

本次调整为：

- 保留本地 `docs/javascripts/config_mathjax.js`
- 在脚本内检测页面是否存在 `.arithmatex`
- 仅当页面真实包含公式节点时，再动态注入 MathJax 主脚本

这样做的目的：

- 保留已有公式渲染能力
- 显著减少无公式页面的 JS 负担
- 让科研文章与普通笔记页按需承担不同的加载成本

### 19.6 修复全站公共脚本的前置异常

在检查加载链路时，发现：

- `docs/javascripts/extra.js`

顶部会无条件执行 `.ml3` 文本动画逻辑。

但当前大多数页面并不包含 `.ml3` 节点，也没有加载 `anime` 依赖，这会导致脚本在页面初始化早期直接报错。

本次已补充：

- 节点存在性判断
- `window.anime` 依赖存在性判断

这样做的目的：

- 避免普通页面在无关动效代码上提前报错
- 保证后续代码折叠等真实通用功能可以稳定执行
- 让全站公共脚本从“历史残留可运行”提升为“对所有页面都安全”
